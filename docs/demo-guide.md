# 面试演示指南

## 演示前准备

```powershell
cd D:\aizhushou
docker compose up -d
docker compose ps
python scripts\smoke_test.py
```

打开：

- 工作台：http://localhost:5010
- API文档：http://localhost:8010/docs
- N8N：http://localhost:5678

## 10分钟演示顺序

### 第1分钟：业务背景

“这个项目不是普通聊天机器人，而是面向绿色咨询顾问的AI辅助交付工作台。它把企业材料转成可解释评分、风险依据和90天行动计划。”

### 第2至4分钟：完整资料案例

上传 `demo/01-complete-company.txt`，展示：

- 自动抽取画像；
- 原文证据和置信度；
- 五维确定性评分；
- 风险后的官方知识引用；
- PDF报告。

### 第5分钟：资料缺失案例

上传 `demo/02-missing-information.txt`，强调：

- 缺失字段显示“待确认”；
- 系统不会把未提供资料直接判定为不合规；
- 低依据风险进入人工审核。

### 第6至7分钟：冲突案例

在同一项目上传：

- `demo/03-conflict-a.txt`
- `demo/03-conflict-b.txt`

展示碳盘查事实冲突、两份原文和字段被评分隔离。

### 第8分钟：工程可靠性

展示页面顶部：

- 本地/Dify模式；
- 成功率和耗时；
- MySQL、Redis状态；
- 最近任务来自Redis或MySQL。

### 第9分钟：评测

点击“运行30条评测”，说明评测覆盖抽取、RAG、缺失、冲突和报告。明确30/30只代表固定基线，不代表生产准确率。

### 第10分钟：技术取舍

总结三点：

1. LLM理解，代码评分；
2. 所有结论尽量有企业证据和知识来源；
3. Dify、知识库、Redis均有降级，不让单点故障中断交付。

## 备用方案

如果模型或网络不可用，使用本地模式完成演示；如果现场Docker异常，准备PDF报告和界面截图。不要临场修改密钥或演示未经测试的新资料。
